Data Fetch and Transmission from Heterogeneous Make Control System using NOXVIEW

Bachelor's Thesis - SCADA System Integration

  1. Home
  2. Projects
  3. Bachelor Thesis

Project Information

  • Category: Bachelor Thesis
  • Institution: Dharmsinh Desai University
  • Year: 2022
  • Technology: PLC, SCADA, LabVIEW, Modbus

Project Overview

Configuration and troubleshooting of NoxVIEW SCADA configurator which is connected with LabVIEW that can fetch and interface data of different PLC hardware devices and networks using software drivers such as Serial Modbus master, TCP IP Modbus master, Mitsubishi TCPIP, Siemens S7 TCPIP etc. through CSV, ODBC, NoxVIEW-IIOT, AWS gateways.

Key Features

  • Heterogeneous System Integration: Unified interface for multiple PLC and control system manufacturers
  • Multi-Protocol Support: Support for various industrial communication protocols
  • Data Gateway Integration: Multiple data transmission gateways (CSV, ODBC, IIOT, AWS)
  • LabVIEW Integration: Seamless connection with LabVIEW for data processing
  • Real-time Monitoring: Continuous data fetching and monitoring capabilities

Supported Hardware & Protocols

PLC Manufacturers

  • Mitsubishi: TCPIP communication protocol
  • Siemens: S7 TCPIP protocol
  • Allen Bradley: Modbus communication
  • Modbus Devices: Serial and TCP/IP Modbus master support

Communication Protocols

  • Serial Modbus Master: RS-232/RS-485 communication
  • TCP IP Modbus Master: Ethernet-based Modbus communication
  • Mitsubishi TCPIP: Native Mitsubishi PLC communication
  • Siemens S7 TCPIP: Native Siemens PLC communication

Data Transmission Gateways

  • CSV Gateway: Data export to CSV format for analysis
  • ODBC Gateway: Database connectivity for data storage
  • NoxVIEW-IIOT: Industrial Internet of Things gateway
  • AWS Gateway: Cloud integration with Amazon Web Services

Technical Implementation

  • SCADA System: NoxVIEW configurator
  • Programming Environment: LabVIEW for data processing
  • Configuration: System configuration and troubleshooting
  • Data Interface: Real-time data fetching and transmission

Project Objectives

  • Create a unified system for interfacing with multiple PLC manufacturers
  • Enable seamless data fetching from heterogeneous control systems
  • Implement multiple data transmission methods for flexibility
  • Provide reliable and real-time industrial data monitoring
  • Integrate with modern cloud and IIOT platforms

Applications

This system has applications in:

  • Industrial automation and monitoring
  • Manufacturing process control
  • SCADA system integration
  • Industrial data analytics
  • Cloud-based industrial monitoring

Key Contributions

  • Unified interface for heterogeneous control systems
  • Multi-protocol support for industrial communication
  • Flexible data transmission gateway implementation
  • Integration with modern cloud and IIOT platforms
